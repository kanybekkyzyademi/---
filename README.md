1. Comparator не үшін керек?

Comparator — бұл объектілерді салыстырудың сыртқы механизмі. Яғни, ол белгілі бір классқа өзгеріс енгізбей-ақ, сол кластың объектілерін әртүрлі тәсілдермен салыстыруға мүмкіндік береді.

Мысалы, Student класын алайық. Бір жағдайда студенттерді GPA бойынша сұрыптау керек болуы мүмкін, ал басқа жағдайда олардың аты немесе жалпы балы бойынша сұрыптау қажет болуы мүмкін. Осындай кезде Comparator қолданылады.

Comparator интерфейсінде compare() әдісі болады. Бұл әдіс екі объектіні салыстырады:

* Егер бірінші объект екіншіден кіші болса — теріс сан қайтарады
* Егер тең болса — 0 қайтарады
* Егер үлкен болса — оң сан қайтарады

Comparator-дың негізгі артықшылықтары:

* Бір классты әртүрлі критерий бойынша сұрыптауға болады
* Кодты өзгерту қажет емес (икемділік жоғары)
* Бірнеше Comparator жасап, әртүрлі логика қолдануға болады



2. Сұрыптау қалай жүзеге асады?

Сұрыптау — бұл элементтерді белгілі бір тәртіпке келтіру процесі. Java тілінде сұрыптау дайын әдістер арқылы жүзеге асады, мысалы:

* Collections.sort()
* List.sort()

Сұрыптау процесі келесі кезеңдерден тұрады:

1. Алдымен студенттердің тізімі (List) құрылады
2. Әр студент объект ретінде сақталады
3. Сұрыптау критерийі беріледі (Comparator немесе Comparable)
4. Java ішкі алгоритмдерді қолданып элементтерді қайта орналастырады

Java көбінесе TimSort алгоритмін қолданады. Бұл алгоритм өте тиімді, себебі:

* Ішінара сұрыпталған массивтермен жақсы жұмыс істейді
* Уақыт күрделілігі орта есеппен O(n log n)

Мысалы, егер GPA бойынша сұрыптасақ:

* Ең жоғары GPA бар студент бірінші тұрады
* Ең төмен GPA бар студент соңында орналасады



3. Қай әдіс тиімді?

Comparable мен Comparator-ды салыстырып қарастырайық.

Comparable:

* Класс ішінде анықталады
* Бір ғана сұрыптау логикасын береді
* Қарапайым жағдайларда тиімді
* compareTo() әдісі арқылы жұмыс істейді

Comparator:

* Класс сыртында анықталады
* Бірнеше сұрыптау тәсілін жасауға мүмкіндік береді
* Икемділігі жоғары
* compare() әдісі арқылы жұмыс істейді

Тиімділік тұрғысынан:

* Егер тек бір ғана критерий қажет болса, Comparable жеткілікті
* Егер жоба күрделі болып, бірнеше сұрыптау керек болса, Comparator әлдеқайда тиімді

Қазіргі программалауда көбінесе Comparator жиі қолданылады, себебі ол масштабталатын және икемді шешім береді.
